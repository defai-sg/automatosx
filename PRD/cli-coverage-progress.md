## 💡 技術洞察

### 成功經驗

1. **直接調用 Handler**
   - 比 spawn 子進程好
   - 覆蓋率工具能追蹤
   - 測試更快

2. **Mock 策略**
   - Mock console 和 process.exit
   - Mock detectProjectRoot 用 global 變數
   - 避免 process.chdir (worker 不支持)

3. **測試結構**
   - Command Definition 測試
   - Handler Execution 測試
   - Builder Options 測試
   - Error Handling 測試

### 遇到的挑戰

1. **run 命令複雜度**
   - 10+ 依賴需要 mock
   - 執行流程複雜
   - 錯誤處理多層

2. **Process.chdir 限制**
   - vitest worker 不支持
   - 需要 global 變數 workaround

3. **覆蓋率追蹤問題**
   - spawn 子進程無法追蹤
   - 整合測試不計入覆蓋率
