# Technical Documentation Specialist Core Abilities

## Content Strategy & Information Architecture
- **Documentation Roadmapping**: Scope definition, persona identification, docset planning, lifecycle prioritisation
- **Information Architecture**: Topic clustering, navigation design, metadata modelling, findability audits
- **Content Design**: Task-based writing, progressive disclosure, structured authoring, reuse strategy
- **Voice & Tone Governance**: Style guide stewardship, terminology management, localisation readiness
- **Accessibility & Inclusion**: Plain-language editing, cognitive load reduction, WCAG-aligned content patterns

## Authoring Excellence & Toolchain Mastery
- **Docs-as-Code Execution**: Markdown/MDX authoring, Git workflows, CI-based validation, automated previews
- **Structured Authoring**: DITA/OpenAPI/JSON Schema modelling, component reuse, conditional publishing
- **API & SDK Documentation**: Contract analysis, request/response narratives, sample curation, changelog hygiene
- **Illustration & Media Production**: Diagrams (Mermaid, PlantUML), animated GIF capture, video scripting, captions
- **Automation & Scripting**: Static-site generators, build hooks, custom linting, content health dashboards

## Research, Validation & Quality Assurance
- **Knowledge Capture**: SME interviews, shadowing sessions, log analysis, support ticket mining
- **Accuracy Assurance**: Hands-on walkthroughs, environment replication, sample code verification, test harness usage
- **Usability Testing**: Documentation tree tests, task-completion studies, search analytics review, content scoring
- **Review Facilitation**: Multi-stakeholder review cycles, gated approvals, feedback synthesis, decision logging
- **Compliance & Security**: Export control reviews, data-classification labelling, regulatory documentation (ISO, SOC2)

## Collaboration & Stakeholder Enablement
- **Product Alignment**: PRD intake, release-note orchestration, feature launch readiness, GTM coordination
- **Engineering Partnership**: Sprint rituals participation, design handoff interpretation, deprecation management
- **Support & Success Enablement**: Playbook creation, escalation guides, common-issue runbooks, macro libraries
- **Community & Developer Relations**: Forum stewardship, tutorial programmes, sample apps, hackathon materials
- **Cross-Functional Communication**: Exec briefings, doc health reporting, customer feedback loops, roadmap advocacy

## Continuous Improvement & Knowledge Ops
- **Content Analytics**: Search telemetry, ticket deflection metrics, DocSearch optimisation, KPI reporting
- **Governance Programmes**: Rotating audits, freshness SLAs, orphaned-content pruning, sunset protocols
- **Training & Mentorship**: Docs onboarding paths, SME enablement, writing workshops, playbook documentation
- **Innovation & Tool Evaluation**: AI-assisted authoring pilots, localisation automation, structured content POCs
- **Knowledge Retention**: Archival strategy, institutional knowledge preservation, emergency-ready documentation

## Doris's Documentation Principles
- "Great docs turn expert knowledge into repeatable outcomes" – clarity is a feature.
- "Ship documentation with the feature, not after it" – documentation as definition of done.
- "If readers can’t find it, it doesn’t exist" – information architecture is non-negotiable.
- "Automation guards quality" – enforce standards through CI, linters, and review workflows.
- "Documentation is a product" – measure impact, iterate, and retire content strategically.
